Projects

Project 1:-Website Status Checker

Concept: A clean, single-page web application where a user can enter any website URL and instantly check if it's online or offline.


​Goal: To build a practical tool that provides real-time website availability status and basic diagnostic information, while learning about front-end to back-end communication (API calls).


​Key Features:

​Front-End: A simple interface with an input box for the URL and a button. The results area will dynamically update to show the status (e.g., "Up" or "Down"). Built with HTML, CSS, and JavaScript.

​Back-End: A simple API (can be built with Node.js/Express or Python/Flask) that receives the URL from the front-end.


​Logic: The back-end sends a request to the target URL. Based on the HTTP response code (e.g., 200 OK vs. 404 Not Found or a timeout), it determines the status and sends a clear success or failure message back to the front-end to be displayed.

Project 2:-Interactive Community Map

Concept: A web application that displays a map where users can collaboratively add and view pins for local points of interest, like a good street food vendor, a quiet park, or a local event.


​Goal: To create a crowd-sourced, interactive map and gain experience with third-party APIs (like a mapping library) and database management.


​Key Features:

​Front-End: Integrates a mapping library like Leaflet.js (free and open-source) or Google Maps API. Users can click on the map to add a pin, which opens a form to add a title and a short description.

​

Back-End: An API to handle creating and retrieving pin data.

​Database: A database (like MongoDB or PostgreSQL) to store the geographic coordinates (latitude, longitude) and text information for each pin. When the page loads, the app fetches all pin data from the database and displays them on the map.
